ASP.NET Core / Async Case Study #1

Async Deadlock via .Result / .Wait()

The problem: In a legacy application, synchronous calls into asynchronous methods (sync-over-async) blocked the thread pool. Under high load, this caused the synchronization context to deadlock, freezing every subsequent HTTP request.

Faulty code (blocking)
✕ Anti-pattern: forces synchronous blocking
public IActionResult GetData()
{
    // .Result blocks the thread until the task completes
    var data = FetchDataAsync().Result; 
    return Ok(data);
}
Optimized code (asynchronous)
✓ Best practice: non-blocking via async/await
public async Task<IActionResult> GetDataAsync()
{
    // Releases the thread back to the pool while waiting on I/O
    var data = await FetchDataAsync(); 
    return Ok(data);
}

The solution: The entire call chain was consistently converted to native asynchronicity (`async/await`). As a result, waiting threads no longer block valuable system resources.

P99 latency after fix: < 120 ms
Entity Framework Core Case Study #2

N+1 Query Problem with EF Core

The problem: With lazy loading enabled, a query for 1,000 records silently generated 1,001 separate SQL statements, because related entities were fetched individually inside a loop. The database connection was completely overloaded.

Faulty code (lazy loading)
✕ Generates 1,001 queries (runtime: 4.2s)
// Loads only customers; related orders are fetched one by one inside the loop
var customers = _context.Customers.ToList(); 
foreach(var c in customers) {
    Console.WriteLine(c.Orders.Count); 
}
Optimized code (eager loading)
✓ Generates exactly 1 query via JOIN (runtime: 38ms)
// Loads data upfront via SQL JOIN and disables EF change tracking for max speed
var customers = _context.Customers
    .Include(c => c.Orders)
    .AsNoTracking()
    .ToList();

The solution: Switched to eager loading via `.Include()`. The additional `.AsNoTracking()` saves CPU cycles and memory, since read-only data no longer needs to be tracked by the Entity Framework context.

Performance gain: 110× faster
